# YourCoffee AI — MVP Telegram Mini App

**Дата:** 2026-05-15
**Статус:** черновик (на ревью)
**Контекст:** курсовая работа, одноразовый проект без долгосрочной поддержки. Цель — рабочий демо-флоу, который можно показать преподавателю в Telegram.

## Цель MVP

Показать живой пользовательский сценарий гостя кофейни: выбор напитка → корзина → заказ → начисление бонусов и просмотр профиля. Без реального бэкенда, без реального ИИ, без оплаты.

**Вне scope MVP** (будет в презентации, но не в коде):
- Реальные ИИ-рекомендации, интеграция с погодой
- Бонус-рулетка, вызов дня, ачивки, многоуровневая геймификация
- Подарок бонусов другому пользователю
- Бэк-офис (бариста-трекер, дашборд владельца, аналитика)
- QR-код для бариста
- Push-уведомления «спящим» клиентам
- Реальная оплата через Telegram Payments

На главном экране эти блоки присутствуют как **статичные UI-элементы** (повторяют фигму), но не функциональны: ИИ-рекомендация — захардкоженный текст, бонус-рулетка и вызов дня — некликабельные карточки.

## Архитектура

Frontend-only SPA, всё состояние в браузере. Запускается внутри Telegram WebView, HTTPS обязателен.

```

              ▲
```

### Стек

- **Vite + React 18 + TypeScript** — каркас SPA
- **react-router-dom** v6 — роутинг
- **zustand** + `persist`-middleware — глобальное состояние с автосохранением в localStorage
- **@telegram-apps/sdk-react** — обёртка над `window.Telegram.WebApp`
- **CSS Modules** — стили, без UI-библиотек
- **Vercel** — хостинг (`vercel.json` минимальный, деплой через `vercel --prod` или GitHub-интеграцию)

### Почему не альтернативы

- **Next.js / SSR**: оверкилл, Telegram открывает как клиентский сайт.
- **`@telegram-apps/telegram-ui`**: готовая UI-либа в стиле Telegram, плохо ляжет на кастомную палитру из Figma.
- **Tailwind**: тоже валиден, но для 5 экранов CSS Modules дают точнее контроль над посадкой по фигме.
- **Свой бэкенд**: лишняя сложность, отдельный хостинг, не нужен для демо.

## Доменная модель

```ts
// menu.json
type MenuItem = {
  id: string;
  category: 'coffee' | 'dessert' | 'other';
  name: string;
  description: string;
  price: number;        // в рублях
  image: string;        // путь к ассету
  addons?: Addon[];     // молоко, сиропы — опционально
};

type Addon = { id: string; name: string; price: number };

// cartStore
type CartItem = {
  menuItemId: string;
  quantity: number;
  selectedAddons: string[];
};

type CartState = {
  items: CartItem[];
  add(item: CartItem): void;
  remove(menuItemId: string): void;
  updateQuantity(menuItemId: string, qty: number): void;
  clear(): void;
  totalPrice(): number;
};

// userStore
type Order = {
  id: string;          // uuid
  createdAt: string;   // ISO
  items: CartItem[];
  total: number;       // в рублях
  bonusEarned: number;
};

type Level = 'Новичок' | 'Любитель' | 'Кофеман';

type UserState = {
  bonusBalance: number;
  totalSpent: number;            // сумма всех заказов в рублях
  orderHistory: Order[];
  level(): Level;                // computed: <500 Новичок, <2000 Любитель, иначе Кофеман
  completeOrder(items, total): void;  // добавляет заказ, начисляет 10%
};
```

## Экраны

### 1. Главная (`/`)
Из фигмы (`node-id=4-41`): шапка с логотипом + переключателем ролей, карточка приветствия с именем из Telegram, ИИ-рекомендация (статика), две плитки (рулетка + вызов дня — некликабельные), карточка бонусов с уровнем, нижняя навигация.

**Динамика:** имя — `Telegram.WebApp.initDataUnsafe.user.first_name` (fallback `Гость`); баланс и уровень — из `userStore`. Кнопка «Заказать» в ИИ-блоке ведёт на `/menu`.

**Переключатель ролей** в шапке (Гость/Бариста/Владелец) — рендерим как чисто декоративный UI-элемент по фигме, без обработчика клика. Функциональность ролей (отдельные экраны для бариста и владельца) выходит за scope MVP.

### 2. Меню (`/menu`)
Список товаров с фильтром по категориям (табы Кофе / Десерты / Другие). Карточки по сетке 2 колонки: картинка, название, цена. Клик → `/menu/:id`.

### 3. Карточка товара (`/menu/:id`)
Большая картинка, название, описание, цена, чекбоксы добавок (если есть), счётчик количества, кнопка «Добавить в корзину» (нативный `MainButton` от Telegram). После добавления — возврат на `/menu`.

### 4. Корзина (`/cart`)
Список добавленного с возможностью менять количество/удалять, итоговая сумма. `MainButton`: «Оформить за N ₽». По клику — добавляет заказ в историю, начисляет 10% бонусов, очищает корзину, показывает экран успеха (модалка/тост), переход на `/profile`.

### 5. Профиль (`/profile`)
Имя и фото из Telegram, баланс бонусов, уровень, история заказов (список карточек: дата, состав, сумма, +бонусы).

### Общее: нижняя навигация
Sticky footer с 4 иконками (Главная / Меню / Корзина / Профиль). Активная вкладка подсвечивается жёлтым `#fab005`. На страницах с `MainButton` тот появляется над nav.

## Бизнес-логика

- **Начисление бонусов:** `10%` от суммы заказа в рублях, округление вниз. Хранится отдельно в `bonusBalance`.
- **Списание бонусов в MVP:** не делаем. Бонусы только копятся (упрощение). При желании можно добавить чекбокс «Оплатить бонусами» на чекауте — но это пункт «можно после MVP».
- **Уровень:** считается из `totalSpent`. `<500` — Новичок, `[500, 2000)` — Любитель, `>=2000` — Кофеман. Это computed-значение, не хранится отдельно.
- **Имя пользователя:** берём из Telegram SDK один раз при инициализации, кешируем в `userStore`. Если SDK недоступен (dev в обычном браузере) — используем мок «Алексей».

## Интеграция с Telegram

В `src/lib/telegram.ts`:
- Инициализация: `WebApp.ready()`, `WebApp.expand()` для полноэкранного режима.
- Тема: читаем `themeParams` и применяем, но фон оставляем фирменный (`#fff7e3`) — кофейный, а не серый из Telegram.
- Имя/фото пользователя: `initDataUnsafe.user`.
- `BackButton`: показываем на всех экранах кроме `/`, по клику — `navigate(-1)`.
- `MainButton`: на `/menu/:id` («Добавить в корзину»), на `/cart` («Оформить»).
- Haptic: лёгкая вибрация при добавлении в корзину и оформлении.

## Деплой

1. Аккаунт на Vercel (через GitHub-логин).
2. Репозиторий → подключаем к Vercel → автодеплой на каждый push в `main`.
3. Получаем `https://your-coffee-ai.vercel.app`.
4. `@BotFather` → `/newbot` → `/setmenubutton` → URL мини-аппы.
5. Открываем бота в Telegram → кнопка меню запускает приложение.

Цена: 0 ₽. Лимиты Vercel free-tier с большим запасом покрывают курсовую.

## Build sequence

1. **Скаффолд + каркас** — Vite-проект, роутер, Telegram SDK, нижняя навигация, заглушки 5 экранов, темизация (CSS-переменные), деплой на Vercel. Бот привязан, приложение открывается из Telegram.
2. **Меню + карточка товара** — `menu.json` со ~10-15 позициями, страница списка с табами категорий, страница деталей с добавками и счётчиком.
3. **Корзина + чекаут** — `cartStore`, экран корзины, поток оформления (создание заказа, начисление бонусов, очистка корзины, переход на профиль).
4. **Профиль** — `userStore`, отображение имени/фото из Telegram, баланс, уровень, история заказов.
5. **Главная + полировка** — статичная ИИ-рекомендация, плитки (рулетка/вызов дня), блок бонусов, переключатель ролей (UI), проверка флоу на реальном устройстве, мелкие правки по фигме.

Каждый шаг = отдельный план + отдельный набор задач, которые мы будем делать последовательно.

## Риски и допущения

- **Telegram SDK в dev-режиме** — открытие в Chrome не даст `WebApp.initData`. Используем мок-пользователя, проверять надо в реальном Telegram.
- **iOS / Android отличия** — WebView ведёт себя чуть по-разному (особенно `100vh`). Это решается на этапе полировки, не закладываем заранее.
- **Картинки товаров** — нужны минимум 10-15 фото кофе/десертов. Источники: экспорт из фигмы (если есть), бесплатные стоки (Unsplash), либо плейсхолдеры с цветным фоном и иконкой.
- **localStorage чистится** при ручной очистке кеша в Telegram — приемлемо для курсовой.
